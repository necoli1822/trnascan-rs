# Error Handling Test Cases
# These test cases validate error detection and handling in the tRNAscan-SE pipeline

## Error Case 1: Empty Input File
### Scenario:
- Input file exists but is empty (0 bytes)

### Expected Error:
- "No sequences found in input file"
- Exit code: 1

### Error Message Format:
FATAL: No sequences found in <filename>

## Error Case 2: Corrupted FASTA File
### Scenario:
- File contains binary/non-text data
- File is truncated mid-sequence

### Expected Error:
- "Invalid FASTA format"
- Exit code: 1

### Error Message Format:
FATAL: Invalid FASTA format in <filename> at line <N>

## Error Case 3: Insufficient Permissions
### Scenario:
- Input file is not readable
- Output directory is not writable

### Expected Error:
- "Permission denied"
- Exit code: 1

### Error Message Format:
FATAL: Cannot read input file <filename>: Permission denied
FATAL: Cannot write to output file <filename>: Permission denied

## Error Case 4: Disk Space Exhausted
### Scenario:
- No space left on device during output writing

### Expected Error:
- "No space left on device"
- Cleanup of partial output files
- Exit code: 1

### Error Message Format:
FATAL: Cannot write output: No space left on device

## Error Case 5: Invalid Score Cutoff
### Scenario:
- Score cutoff is negative
- Score cutoff is not a number

### Expected Error:
- "Invalid score cutoff value"
- Exit code: 1

### Test Commands:
tRNAscan-SE -E -X -10 input.fa
tRNAscan-SE -E -X abc input.fa

### Error Message Format:
FATAL: Invalid score cutoff: <value>

## Error Case 6: Invalid Padding Value
### Scenario:
- Padding value is negative
- Padding value is not an integer

### Expected Error:
- "Invalid padding value"
- Exit code: 1

### Test Commands:
tRNAscan-SE -E -z -5 input.fa
tRNAscan-SE -E -z 3.14 input.fa

### Error Message Format:
FATAL: Invalid padding value: <value>

## Error Case 7: Output File Already Exists (No Force)
### Scenario:
- Output file already exists
- -Q (force overwrite) flag not specified
- Interactive mode: should prompt user
- Non-interactive mode: should error

### Expected Behavior:
- Prompt: "File <filename> exists. Overwrite? [y/N]"
- If 'n' or non-interactive: Exit with error
- Exit code: 1 (if not overwriting)

## Error Case 8: Missing Required Model Files
### Scenario:
- Covariance model files not found in expected location

### Expected Error:
- "Cannot find model file"
- Exit code: 1

### Error Message Format:
FATAL: Unable to open <model_file> covariance model file

## Error Case 9: Invalid Mitochondrial Model
### Scenario:
- Unknown mitochondrial model specified

### Expected Error:
- "Invalid mitochondrial model"
- Exit code: 1

### Test Command:
tRNAscan-SE -M invalid_model input.fa

### Error Message Format:
FATAL: Invalid mitochondrial model: <model_name>
Valid options: mammal, vert

## Error Case 10: Sequence Too Long
### Scenario:
- Input sequence exceeds maximum allowable length

### Expected Behavior:
- Warning: "Sequence <name> is very long (<N> bp)"
- Continue processing or skip with warning
- Do not crash

### Warning Message Format:
WARNING: Sequence <name> is exceptionally long (<N> bp) and may take significant time to process

## Error Case 11: No tRNAs Found
### Scenario:
- Input file is valid but contains no tRNA sequences
- All sequences fail score cutoff

### Expected Behavior:
- Complete successfully
- Output: "No tRNAs found" in results
- Exit code: 0 (not an error, just no results)

### Output Format:
No tRNAs found in <filename>

## Error Case 12: Memory Allocation Failure
### Scenario:
- System runs out of memory during processing

### Expected Error:
- "Memory allocation failed"
- Cleanup and graceful exit
- Exit code: 1

### Error Message Format:
FATAL: Memory allocation failed while processing <sequence_name>

## Error Case 13: Invalid Configuration File
### Scenario:
- Configuration file specified with -c but not found
- Configuration file has invalid format

### Expected Error:
- "Cannot open configuration file"
- Exit code: 1

### Test Command:
tRNAscan-SE -c nonexistent.conf -E input.fa

### Error Message Format:
FATAL: Cannot open configuration file: <filename>

## Error Case 14: Infernal/CMSEARCH Not Found
### Scenario:
- Required external binaries (cmsearch, etc.) not in PATH
- Binary paths in config are incorrect

### Expected Error:
- "Cannot find required program"
- Exit code: 1

### Error Message Format:
FATAL: Cannot execute <program_name>: command not found
Check PATH or configuration file settings

## Error Case 15: Temporary Directory Not Writable
### Scenario:
- Temp directory specified in config doesn't exist
- Temp directory has no write permission

### Expected Error:
- "Cannot write to temporary directory"
- Exit code: 1

### Error Message Format:
FATAL: Cannot write to temporary directory: <temp_dir>

## Error Case 16: Signal Interrupt (SIGINT)
### Scenario:
- User presses Ctrl+C during execution

### Expected Behavior:
- Clean up temporary files
- Exit gracefully
- Exit code: 130 (128 + SIGINT signal number)

### Message Format:
Interrupted by user. Cleaning up...

## Error Case 17: Invalid Match Pattern
### Scenario:
- --match option with invalid regex pattern

### Expected Error:
- "Invalid match pattern"
- Exit code: 1

### Test Command:
tRNAscan-SE -E --match "[invalid(" input.fa

### Error Message Format:
FATAL: Invalid match pattern: <pattern>

## Error Case 18: Conflicting Output Options
### Scenario:
- Both --detail and --brief specified
- Incompatible output format combinations

### Expected Error:
- "Conflicting options"
- Exit code: 1

### Test Command:
tRNAscan-SE -E --detail --brief input.fa

### Error Message Format:
FATAL: Conflicting options: --detail and --brief cannot be used together

## Error Case 19: Invalid GFF/BED Output Path
### Scenario:
- GFF/BED output path is a directory, not a file
- Output path contains invalid characters

### Expected Error:
- "Invalid output path"
- Exit code: 1

### Error Message Format:
FATAL: Invalid output file path: <path>

## Error Case 20: Numerical Overflow
### Scenario:
- Extremely large score values
- Position values exceeding integer limits

### Expected Behavior:
- Detect overflow
- Report error or clamp to maximum value
- Do not silently produce incorrect results

### Error Message Format:
WARNING: Numerical overflow detected in <context>: value clamped to maximum
