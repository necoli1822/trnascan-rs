# Pipeline Input Validation Test Cases
# These test cases validate input handling for the tRNAscan-SE pipeline

## Test Case 1: Valid FASTA Input (Single Sequence)
### Input:
>CELF22B7
GCACGGATGGCCGAGTGGTCTAAGGCGCCAGACTCAAGCGAAATGCTTGCCTCATGCTCGAGGTCGACTGGGTGTTCTGGTACTCGTATGGGTGCGTGGGTTCGAATCCCACTTCGTGCA

### Expected Behavior:
- Successfully parse sequence
- Detect sequence name: CELF22B7
- Process sequence for tRNA detection

## Test Case 2: Valid FASTA Input (Multiple Sequences)
### Input:
>seq1
ATCGATCGATCGATCG
>seq2
GCTAGCTAGCTAGCTA

### Expected Behavior:
- Parse both sequences
- Process each independently
- Maintain sequence order in output

### Input:
(empty file)

### Expected Behavior:
- Error: No sequences found

## Test Case 5: FASTA with Whitespace in Sequence
### Input:
>test_seq

ATCG

### Expected Behavior:
- Strip whitespace

## Test Case 6: FASTA with Invalid Characters
### Input:
>invalid_seq

### Expected Behavior:

## Test Case 7: FASTA with Very Long Sequence Name

### Input:

### Expected Behavior:
- Handle special characters in sequence name
